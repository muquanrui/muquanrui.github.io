类的定义

类之间的关系

主要包括：关联，聚集，组合，泛化，依赖。

关联（association）

模型元素中一种语义联系，具有共同的结构特性，行为特性，关系和语义的描述。

聚集

用来描述两个类之间的整体——部分关系，其中一个类为整体，由一个或多个部分分类组成，聚合中，部分可以没有整体类而存在。

组合

一种特殊的聚合关联，在组合关联中用来组成整体类的部分类不能独立存在。销毁整体类的时候会同时销毁部分类。

泛化

继承关系，指一般元素和特殊元素之间的分类关系。

依赖

抽象类和接口

边界类、控制类和实体类

  • 边界类，boundary class
  • 控制类，control class
  • 实体类，entity class

边界类

边界类位于系统与外界的交界处。

  • 用户界面类，如窗口，对话框
  • 通信协议类，如TCP/IP
  • 直接与外部设备交互的类
  • 直接与外部系统交互的类

表示方法：

通过用例图可以确定需要的边界类，每个actor至少对应一个边界类。

实体类

实体类保存要存放进存储体的信息。

实体类通过时间流和交互图发现，采用目标域术语命名。

通常实体类对应数据库中的表，其属性对应表的字段，但实体类与数据库表并不是一一对应。

控制类

控制类负责管理和控制其他类的工作。

每个用例通常有一个控制类，控制用例中的时间顺序，控制类也可以在多个用例间共用。控制较少接受信息，发出较多消息。

类图

类加上它们之间的关系构成类图。其中包含接口、包、关系等元素，也可以包含对象、链等实例。描述类和类之间的静态关系。
