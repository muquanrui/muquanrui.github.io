8.1 为什么要使用泛型程序设计

泛型程序设计（Generic programming）意味着编写的代码可以被很多不同类型的对象所重用。

8.1.1 类型参数的好处

泛型提供了类型参数（type parameter）：ArrayList<String> files = new ArrayList<String>()

调用get时，不需要进行强制类型转换：String filename = files.get(0);

添加对象时，编译器会检查对象类型：files.add(new File("..."));

类型参数可以使程序具有更好的可读性和安全性。

8.1.2 谁想成为泛型程序员

实现一个泛型类并不容易，一个泛型程序员的任务就是预测出所有类的未来可能有的所有用途。

8.2 定义简单泛型类

一个泛型类（generic class）就是具有一个或多个类型变量的类。

public class Pair<T>{
    private T first;
    private T second;

    public Pair(){
        first = null;
        second = null;
    }

    public Pair(T first, T second){
        this.first = first;
        this.second = second;
    }

    public T getFirst(){
        return first;
    }

    public T getSecond(){
        return second;
    }

    public void setFirst(T newValue){
        first = newValue;
    }

    public void setSecond(T newValue){
        second = newValue;
    }
}

Pair类引入了一个类型变量T并放在类名后面，泛型类还可以有多个类型变量，比如public class Pair<T, U>{...}，类定义中的类型变量指定方法的返回类型以及域和局部变量的类型。

注意：在Java中，一般使用变量E表示集合的元素类型，K和V分别表示表的关键字与值的类型，T（U、S）表示任意类型。

用具体的类型替换类型变量就可以实例化泛型类型，如：Pair<String>。

8.3 泛型方法

泛型方法是带有泛型变量的方法，可以定义在泛型类和普通类中。

class ArrayAlg{
    public static <T> T getMiddle(T... a){
        return a[a.length / 2];
    }
}

定义时，类型变量<T>要放在修饰符后面，返回类型的前面。

调用时，具体类型放在方法名前的尖括号中（当编译器可以推断类型时，方法调用可以省略类型参数）：String middle = ArrayAlg.<String>getMiddle("John", "Q", "Public");

8.4 类型变量的限定

class ArrayAlg{
    public static <T> T min(T[] a){
        if(a == null || a.length == 0)
            return null;
        T smallest = a[0];
        for(int i=1; i<a.length; i++){
            if(smallest.compareTo(a[i]) > 0)
                smallest = a[i];
        }
        return smallest;
    }
}

变量smallest类型为T，即它可以是任何一个类的对象，但是smallest要调用compareTo方法，就必须对T进行限制，即限制T为实现了Comparable接口（只含一个方法compareTo的标准接口）的类。

public static <T extends Comparable> T min(T[] a)...

对于<T extends BoundingType>，表示的是T是绑定类型的子类型（subtype），T的绑定类型可以是类，也可以是接口。

类型变量可以有多个限定，比如T extends Comparable & Serializable。限定类型用&分隔，但是限定类型可以是多个接口，但最多只能有一个类（继承的超类只能是一个），如果限定类型中有类，则它必须是限定列表中的第一个。

8.5 泛型代码和虚拟机

8.5.1 类型擦除

定义一个泛型类型时会自动提供一个相应的原始类型（raw type），原始类型的名字就是删去类型参数后的泛型类型名。擦除（erased）类型变量，并替换为限定类型（无限定的变量使用Object替换）。

比如在Pair类中，所有的<T>都被替换为Object。

8.5.2 翻译泛型表达式

当程序调用泛型方法时，如果擦除返回类型，编译器将插入强制类型转换。

Pair<Employee> buddies = ...;
Employee buddy = buddies.getFirst();

擦除getFirst的返回类型后将返回Object类型，编译器自动插入Employee的强制类型转换。编译器把这个方法翻译为两条虚拟机指令：

  1. 对原始方法Pair.getFirst的调用。
  2. 将返回的Object类型强制转换为Employee类型。

8.5.3 翻译泛型方法

类型擦除也出现在泛型方法中。比如：public static <T extends Comparable> T min(T[] a)...，擦除类型后只剩下：public static Comparable min(Comparable[] a)...。

方法的擦除会带来两个复杂问题。

class DateInterval extends Pair<LocalDate>{
    public void setSecond(LocalDate second){
        if(second.compareTo(getFirst()) >= 0)
            super.setSecond(second);
    }
    ...
}

一个日期区间是一对LocalDate对象，并且需要覆盖这个方法来确保第二个值永远不小于第一个值。这个类擦除后变为：

class DateInterval extends Pair{
    public void setSecond(LocalDate second){...}
    ...
}

此时存在另一个从Pair继承的setSecond方法：public void setSecond(Object second){...}

这显然是一个不同的方法，因为它有一个不同类型的参数，因此不满足继承关系中的方法重写。

DateInterval interval = new DateInterval(...);
Pair<LocalDate> pair = interval;
pair.setSecond(aDate);

这里想对setSecond的调用具有多态性，并调用最合适的那个方法，即DateInterval.setSecond，此时类型擦除与多态产生冲突。解决这个问题需要编译器在DateInterval类中生成一个桥方法（bridge method）。

public void setSecond(Object second){setSecond((Date)second);}

总之，Java泛型转换的事实是：

  • 虚拟机中没有泛型，只有普通的类和方法。
  • 所有的类型参数都用它们的限定类型来替换。
  • 桥方法被合成用来保持多态。
  • 为保持类型安全性，必要时插入强制类型转换。

8.6 约束与局限性

大多数限制由类型擦除引起。

  1. 不能用基本类型实例化类型参数：没有Pair<double>，只有Pair<Double>。
  2. 运行时类型查询只适用于原始类型：a instanceof Pair<String>或a instanceof Pair<T>都只能检查a是否是任意类型的一个Pair，编译器会报错。getClass同理，只能返回原始类型Pair.class。
  3. 不能创建参数化类型的数组：Pair<String>[] table = new Pair<String>[10]是不被允许的。
  4. Varargs警告：@SafeVarargs标注可以消除创建泛型数组的限制。
  5. 不能实例化类型变量：不能使用new T(...)，new T[...]或T.class这样的表达式。
  6. 不能构造泛型数组：T[] mm = new T[2]这样构造泛型数组的表达式不被允许。
  7. 泛型类的静态上下文中类型变量无效：禁止使用带有类型变量的静态域和方法。
  8. 不能抛出或捕获泛型类的实例：public class Problem<T> extends Exception{...}是不合法的。catch子句中也不能使用类型变量。
  9. 可以消除对受查异常的检查：通过使用泛型类、擦除和@Suppress Warnings注解，可以消除Java类型系统的部分基本限制。
  10. 注意擦除后的冲突。

8.7 泛型类型的继承规则

无论S与T有什么联系，Pair<S>与Pair<T>没有什么联系。比如Pair<Manager>绝对不是Pair<Employee>的一个子类。

但是泛型类可以被扩展或实现其他的泛型类，这一点与普通类没有区别。

8.8 通配符

8.8.1 通配符概念

通配符类型中允许类型参数变化。例如Pair<? extends Employee>，表示任何泛型Pair类型，它的类型参数都是Employee的子类，如Pair<Manager>。

8.8.2 通配符的超类型限定

通配符限定与类型变量限定类似，还有一个附加能力，即可以指定一个超类型限定（supertype bound），比如? super Manager，表示限制为Manager类的所有超类型。

8.8.3 无限定通配符

无限定通配符例如Pair<?>，它和Pair的本质不同在于：可以用任意Object对象调用原始Pair类的setObject方法。

8.8.4 通配符捕获

通配符?不能作为一种类型在代码中使用。
